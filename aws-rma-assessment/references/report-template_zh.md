## 下一步建议

对于评分为 Level 1 的领域，建议使用 `aws-resilience-modeling` 进行以下深度分析：

| Level 1 领域 | 建议 Modeling 任务 | 重点分析 |
|-------------|-------------------|---------|
| 灾难恢复 | 任务 2（故障模式）+ 任务 4（业务影响） | DR策略验证和业务影响分析 |
| 高可用性 | 任务 1（组件映射）+ 任务 2（故障模式） | 架构依赖和容错分析 |
| 可观察性 | 任务 1（组件映射）+ 任务 3（韧性评估） | 监控差距识别和评分 |

本评估建议与 `aws-resilience-modeling` 深度分析配合使用，以形成完整的风险缓解闭环。

## 评分对齐参考

| RMA 级别 | 约等于 Modeling 星级 | 描述 |
|---------|-------------------|------|
| Level 1（初始） | 1-2 星 | 无正式流程 |
| Level 2（已定义） | 2.5-3.5 星 | 基本流程已建立 |
| Level 3（优化） | 4-5 星 | 成熟、自动化、持续改进 |

*注意：这是近似映射，两个评估的维度和粒度不同。*

## HTML报告生成（可选）

如果用户需要HTML版本，使用pandoc转换：

```bash
# 检查pandoc是否可用
if command -v pandoc &> /dev/null; then
    pandoc {report-file}.md \
      -f gfm \
      -t html5 \
      --standalone \
      --toc \
      --toc-depth=3 \
      --css=https://cdn.jsdelivr.net/npm/github-markdown-css@5/github-markdown.min.css \
      --metadata title="RMA 韧性评估报告" \
      -o {report-file}.html
fi
```
